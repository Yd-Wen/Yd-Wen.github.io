## 🎯 为什么需要心法

用 AI 做项目时，有时 AI 特别听话一次就能做对，有时又特别倔强怎么说都做不好。这是因为 Vibe Coding 和传统编程一样，都有自己的"心法"——经过无数人验证的思维方式和工作原则。掌握这些心法，能让 AI 更好地理解意图，做出更高质量的产品。

## 📊 五大实践心法

### 一、先想后做，不要边想边做

::: tip
**想清楚再动手，比动手后再想清楚要容易得多**
:::

| 错误做法 | 正确做法 |
|:---|:---|
| 打开工具直接开始写提示词 | 先用纸笔或文档梳理需求 |
| 边做边想，想到哪做到哪 | 明确功能列表和页面结构 |
| 频繁修改方向，反复返工 | 一次性想清楚，减少后期改动 |

**实践建议**：
- 使用 AI 生成需求文档模板
- 画出简单的页面结构图
- 列出功能清单和优先级

---

### 二、小步快跑，不要一步登天

::: tip
**把大任务拆成小任务，逐个击破**
::: 

| 错误做法 | 正确做法 |
|:---|:---|
| "给我做一个电商网站" | 先做一个商品列表页 |
| 一次性生成整个项目 | 先生成框架，再添加功能 |
| 需求过于宏大，AI 理解困难 | 每次只聚焦一个具体功能 |

**实践建议**：
- 使用分步骤提示词："第一步... 第二步..."
- 每个步骤验证通过后再进行下一步
- 及时保存每个阶段的成果

---

### 三、说人话，不要说代码

::: tip
**用自然语言描述需求，而不是技术术语**
::: 

| 错误做法 | 正确做法 |
|:---|:---|
| "使用 React Hooks 实现状态管理" | "点击按钮后数字要增加" |
| "帮我写一个 Redux reducer" | "这个数据要在多个页面共享" |
| 堆砌技术名词，AI 理解偏差 | 描述用户场景和期望效果 |

**实践建议**：
- 描述"用户会做什么"而不是"代码要怎么写"
- 用生活中的比喻来解释需求
- 提供具体的示例和场景

---

### 四、相信 AI，但不要迷信 AI

::: tip
**AI 是助手，不是替代者**
:::

| 过度依赖 | 合理使用 |
|:---|:---|
| AI 生成的代码直接上线 | 人工审查关键逻辑 |
| 完全不懂代码原理 | 理解 AI 生成的基本逻辑 |
| 出现问题就怪 AI 不行 | 学会调试和引导 AI 修正 |

**实践建议**：
- 对核心功能进行人工验证
- 学习基础的代码阅读能力
- 建立"AI 生成 + 人工把关"的工作流

---

### 五、快速试错，不要害怕失败

::: tip
**Vibe Coding 的魅力就在于低成本试错**
:::

| 错误心态 | 正确心态 |
|:---|:---|
| 追求完美，迟迟不敢开始 | 先做出来，再慢慢优化 |
| 一次失败就放弃 | 把失败当作学习机会 |
| 担心浪费 AI 调用额度 | 试错成本远低于传统开发 |

**实践建议**：
- 设置"试错预算"，允许自己失败
- 多尝试不同的提示词和工具
- 记录失败案例，总结经验教训

## 🔧 心法应用实践

### 完整工作流程示例

```plaintext
1. 需求梳理（心法一）
   └── 用纸笔列出功能清单
   
2. 任务拆解（心法二）
   └── 拆分为：框架搭建 → 首页 → 详情页 → 支付功能
   
3. 自然语言描述（心法三）
   └── "用户点击购买按钮后，弹出支付窗口"
   
4. 人工审查（心法四）
   └── 检查支付逻辑的安全性
   
5. 快速迭代（心法五）
   └── 不满意就重新生成，直到满意为止
```

## ⚔️ 心法对比：传统编程 vs Vibe Coding

| 维度 | 传统编程 | Vibe Coding |
|:---|:---|:---|
| **思维方式** | 面向机器，关注语法细节 | 面向需求，关注用户价值 |
| **开发节奏** | 前期设计完善，后期少改动 | 快速试错，持续迭代 |
| **沟通方式** | 写代码（机器语言） | 说人话（自然语言） |
| **错误态度** | 尽量避免错误 | 拥抱试错，快速修正 |
| **人机关系** | 人写代码，机器执行 | 人提需求，AI 生成，人把关 |

## 💡 常见误区与纠正

| 误区 | 纠正 |
|:---|:---|
| "AI 什么都能做，我什么都不用管" | AI 是助手，最终决策和责任在人 |
| "提示词越复杂越好" | 清晰简洁比复杂堆砌更有效 |
| "一次生成完美产品" | 接受迭代，小步快跑 |
| "必须懂技术才能用" | 从说人话开始，技术可以边做边学 |
| "一个工具走天下" | 根据场景选择合适工具组合 |

```plaintext
  │        │        │
  ▼        ▼        ▼
严格遵循  灵活运用  形成直觉
五个心法  心法组合  人AI合一
```

## 📝 关键洞察

::: tip
**Vibe Coding 的心法 = 产品经理思维 + 敏捷开发方法 + 人机协作智慧**
:::

- **心法一** 对应产品规划：需求清晰才能执行高效
- **心法二** 对应敏捷开发：迭代交付，持续反馈
- **心法三** 对应用户思维：关注价值，而非实现
- **心法四** 对应风险管理：信任但验证，授权但监督
- **心法五** 对应创新文化：容忍失败，鼓励尝试
